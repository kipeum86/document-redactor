# document-redactor v1.0.0

_Released YYYY-MM-DD_

## What's new

- First public release. Privacy-preserving in-browser DOCX redactor for Korean/English legal documents.
- Drop a `.docx` file onto the drop zone, review the detected candidates (entity names, aliases, PII), click **Apply and verify** (or ⌘/Ctrl + Enter), and download the redacted copy as `{name}.redacted.docx`.
- **No network. No uploads. No accounts.** The whole tool is one HTML file. Open it in any modern browser and it runs locally — `fetch`, `XMLHttpRequest`, and `WebSocket` are banned at the source level (ESLint), blocked at the bundle level (no `fetch(` token in the output), and forbidden at runtime (`connect-src 'none'` in the CSP meta tag).
- Track-changes flattened, comments stripped, metadata (author, last modified by, title) scrubbed — your redacted document no longer carries editing history or authorship traces.
- Outputs are byte-stable: the same input + the same selections produce a byte-identical `.docx` + SHA-256. Two reviewers hashing the same output get the same digest.

```bash

# expected output:
#   document-redactor.html: OK
```

If you see `OK`, the file is byte-identical to what the author published. Anything else — do not run it.

## Known limitations

- **Level picker is cosmetic.** Only the Standard rule set runs in v1; Conservative and Paranoid are UI stubs and will ship in v1.x.
- **No click-to-select in the document preview.** The preview pane is a placeholder; all candidate review happens through the right-hand panel.
- **View source + Audit log buttons are disabled.** D8.1 (self-hash modal) and D8.2 (audit log table) are planned for v1.x.
- **Layout degrades to 2-column below 720 px.** The 3-column desktop layout requires ≥1024 px for comfortable use.
- **English + Korean contracts tested end-to-end.** Other document types (opinions, briefs, memos) should work — the engine is text-based, not contract-specific — but v1's test fixture is a bilingual NDA.
